We received a few more questions related to CW3. Please read these answers carefully:

1. How do we prove that our code is not AI-generated?
You do not need to worry about disproving AI usage. You just need to be able to explain and justify your code and your approach/algorithms in your written report. If you have written all of your code yourself and did not use AI (beyond non-substantive use), then please do not worry about proving you did not use AI.
 
Remember that minor AI use is permitted for tasks such as grammatical corrections, helping you think about test cases, etc. -- BUT, you need to cite in your written report any use of AI, including which tool(s) you used and how you used them (e.g., the original input, how you prompted the AI, and what you did with the output). If in doubt, please contact us.
 
2. Are we allowed to use sorting algorithms?
To clarify from our previous announcement, you are allowed to use existing sorting algorithms such as Collections.sort(), or implement your own sorting algorithms if you wish; but this is not required for the assignment.
 
3. Are we allowed to use Lambda functions when implementing our GUI?
No, using Lambda functions is forbidden.
 
4. What JavaFX methods, libraries, and techniques are we allowed to use that are within the scope of this course?
You can use any techniques (method, libraries, etc.) described in the Liang textbook, Chapters 14, 15, and 16, except for Lambda functions.
 
5. Can we input our code into an AI tool and ask it to find bugs for us?
No, for two reasons: (1) this is considered "substantial" (i.e., major) use of AI; and (2) it will not work. Just giving AI your code and asking it to find bugs is not an effective way to surface bugs. Rather, the approach you should be using is to put together test cases and try to give your program as many edge cases or edge inputs you can think of that could break your system. You may use AI to help you think about test cases -- but you must cite this usage in your written report.
 
6. The assignment specifications mention that we need to describe our testing methods and results. Should we make and run tests to test our code and the methods we need to complete?
Yes, this is recommended to ensure that your code will run without bugs. You also need to describe your testing methods and results in your written report.
 
7. Can we include pictures (e.g., of the GUI) when describing our testing methods and results?
Yes, this is recommended to increase readability, but also don't forget to pair it with explanation paragraph(s).
 
8. Can we include an appendix in our written report? Would the appendix count toward the 15-page limit?
Yes, you can include an appendix. However, your appendix will count toward your 15-page limit.
 
9. Can we use/import java.io.InputStream?
Yes, this is fine.
