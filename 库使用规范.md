## 🚨 1. The "Zero Marks" Zone (Strictly Forbidden)
**If any of these appear in your code, you risk getting 0 marks or academic misconduct.**

- [ ] **NO Lambda Expressions:** `->` (e.g., in `forEach` or JavaFX events).
- [ ] **NO Streams API:** `.stream()`, `.filter()`, `.map()`, `.collect()`.
- [ ] **NO StringBuilder / StringBuffer:** Must use `+` or `String.join` for concatenation.
- [ ] **NO `java.nio` Package:** Do not use `Files.readAllLines`, `Path`, etc.
- [ ] **NO `var` Keyword:** Must explicitly declare types (e.g., `int`, `String`).
- [ ] **NO Lombok:** No `@Data`, `@Getter` annotations.
- [ ] **NO Custom Interfaces/Abstract Classes:** You can *implement/extend* existing ones (e.g., `Application`), but do not define new ones.
- [ ] **NO `record` or `enum` keywords.**
- [ ] **NO `System.err`:** Use `System.out.println` for errors.
- [ ] **NO External Libraries:** No Maven/Gradle dependencies (except JavaFX).
- [ ] **NO Existing Encryption Libs:** Do not use `java.security.MessageDigest`. Write your own simple hashing logic.

## ✅ 2. Mandatory & Allowed Libraries
**You MUST use these tools covered in the lectures.**

### Data Structures (java.util)
- [ ] **List:** Interface `List<T>`, Implementation `ArrayList<T>` or `LinkedList<T>`.
- [ ] **Map:** Interface `Map<K,V>`, Implementation `HashMap<K,V>` (or `TreeMap`).
- [ ] **Set:** (Optional but allowed) `HashSet<T>`.
- [ ] **Sorting:** `Collections.sort()` is allowed, but not required.

### File I/O (java.io)
- [ ] **Reading:** `Scanner` (recommended) OR `BufferedReader` + `FileReader`.
- [ ] **Writing:** `PrintWriter` OR `BufferedWriter` + `FileWriter`.
- [ ] **File Object:** `File` class for paths.

### Date & Time
- [ ] **Class:** `java.time.LocalDate` / `LocalDateTime`.

### GUI (Advanced Feature)
- [ ] **Library:** **JavaFX** only.
- [ ] **Tools:** Scene Builder, CSS files (`style.css`).
- [ ] **Input Hiding:** `PasswordField` is allowed.

## 🛠 3. Implementation Requirements

### OOP Principles
- [ ] **Encapsulation:** All instance variables must be `private`. Access via `public` Getters/Setters.
- [ ] **Inheritance:** Must use `extends` at least once.
- [ ] **Polymorphism:** Must demonstrate `@Override` (e.g., `toString()`).
- [ ] **Constructors:** Must use `super()` in subclass constructors.

### String Manipulation
- [ ] **Comparison:** Use `.equals()` or `.equalsIgnoreCase()`. **NEVER** use `==`.
- [ ] **Concatenation:** Use `+` operator.
- [ ] **Parsing:** Use `.split(",")` for CSV data.
- [ ] **Formatting:** `String.format()` is allowed.

### Exception Handling
- [ ] **Try-Catch:** Must handle `IOException`, `FileNotFoundException`.
- [ ] **Data Safety:** Handle `NumberFormatException` when parsing IDs or Ratings.
